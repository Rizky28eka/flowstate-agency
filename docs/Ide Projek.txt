# AGENCYFLOW - PLATFORM MANAJEMEN PROYEK & KLIEN TERINTEGRASI
## Dokumentasi Pengembangan Project (Extended Version)

---

## BAGIAN 1: EXECUTIVE SUMMARY

### Ringkasan Proyek
AgencyFlow adalah platform Software as a Service (SaaS) yang dirancang khusus untuk mengatasi fragmentasi alur kerja di software agency dan creative agency skala kecil hingga menengah. Platform ini mengintegrasikan seluruh siklus hidup proyek—dari pembuatan quotation, manajemen tugas, komunikasi klien, hingga invoicing—dalam satu ekosistem terpadu.

### Value Proposition
"Satu Platform, Semua Alur Kerja Agency Anda" - AgencyFlow mengeliminasi kebutuhan akan 5-7 tools berbeda yang biasanya digunakan agency, menghemat waktu hingga 15-20 jam per bulan untuk tugas administratif.

### Target Market
- Software development agencies (5-50 karyawan)
- Creative & design agencies
- Digital marketing agencies
- Freelancer teams & collective
- Estimasi: 50,000+ potential agencies di Indonesia, 500,000+ di Asia Tenggara

## BAGIAN 2: PROBLEM STATEMENT (DEEP DIVE)

### 2.1 Masalah Utama yang Diselesaikan

#### Pain Point #1: Fragmentasi Tools
**Situasi Saat Ini:**
- Quotation: Google Sheets/Excel/Word
- Komunikasi: WhatsApp, Email, Telegram (terpisah-pisah)
- Task Management: Trello, Asana, Notion (tidak terintegrasi dengan klien)
- Time Tracking: Manual atau tools terpisah
- Invoicing: Software akuntansi atau manual
- File Sharing: Google Drive, Dropbox (tersebar)

**Dampak Negatif:**
- Informasi penting terselip di berbagai platform
- Waktu terbuang 2-3 jam/hari untuk switching context
- Risiko kehilangan data historis percakapan
- Onboarding karyawan baru memakan waktu 2-3 minggu

#### Pain Point #2: Lack of Client Transparency
**Situasi Saat Ini:**
- Klien tidak tahu progress real-time
- Pertanyaan berulang: "Sudah sampai mana proyeknya?"
- Trust issues karena kurangnya visibilitas
- Screenshot dan manual update via chat/email

**Dampak Negatif:**
- 30-40% waktu PM habis untuk update status ke klien
- Ketidakpuasan klien meski proyek on-track
- Reputasi agency menurun
- Churn rate klien meningkat

#### Pain Point #3: Inaccurate Project Estimation
**Situasi Saat Ini:**
- Estimasi berbasis "feeling" atau pengalaman
- Tidak ada data historis yang terstruktur
- Scope creep tidak terdokumentasi dengan baik
- Profitabilitas proyek tidak terukur

**Dampak Negatif:**
- 40-50% proyek melebihi budget waktu
- Margin profit terkikis tanpa disadari
- Tim overworked karena salah estimasi
- Sulit melakukan pricing yang kompetitif namun profitable

#### Pain Point #4: Invoicing & Payment Delays
**Situasi Saat Ini:**
- Invoice dibuat manual setelah milestone
- Follow-up pembayaran via chat manual
- Tidak ada sistem reminder otomatis
- Riwayat pembayaran tidak terorganisir

**Dampak Negatif:**
- Cash flow tidak sehat
- Average payment delay: 15-30 hari lebih lama
- Waktu admin terbuang untuk follow-up
- Potensi bad debt meningkat

### 2.2 Data & Statistik Pendukung
- 68% agency menggunakan 4+ tools berbeda (Survey fictional)
- Average waktu administratif: 25-30% dari total working hours
- Client satisfaction meningkat 45% dengan transparency tools
- ROI positif tercapai dalam 3-6 bulan setelah adopsi unified platform

## BAGIAN 3: SOLUTION OVERVIEW

### 3.1 Visi Produk
AgencyFlow menjadi "Operating System" bagi modern agencies, di mana semua aspek bisnis—dari sales hingga delivery dan finance—berjalan dalam satu ekosistem yang seamless.

1. **Simplicity First**: Interface yang intuitif, tidak memerlukan training ekstensif
2. **Client-Centric**: Setiap fitur dirancang dengan mempertimbangkan experience klien

2. **Client Portal as Standard**: Bukan add-on, tapi core feature
3. **Indonesian-First**: UI/UX, language, dan workflow disesuaikan dengan kultur bisnis Indonesia

- **Quotation Form**:
  - Client information (auto-complete dari database klien existing)
  - Project details: Name, Description, Timeline
  - Line items:
    - Drag service dari library atau add manual
    - Quantity adjustment
    - Custom discount (% atau fixed amount)
    - Tax calculation (PPN 11%)
  - Payment terms:

- **Preview & Styling**:
  - Live preview saat edit
  - Upload company logo
  - Brand color customization

**Output Options:**
- PDF download dengan professional formatting
- Share via unique link (trackable)

**Technical Implementation:**
- PDF generation: jsPDF atau Puppeteer (HTML to PDF)

- Rich text editor: TipTap atau Quill

#### 4.2.2 Quotation Tracking & Analytics
**Dashboard Metrics:**
- Total quotations sent (this month/year)
- Conversion rate (Approved / Total Sent)
- Average deal size
- Average time to approval
- Win/Loss analysis

**Individual Quotation Tracking:**
- Status labels: Draft, Sent, Viewed, Approved, Rejected, Negotiating
- View count & last viewed timestamp

- Reminder system untuk follow-up

**Integration:**
- Email tracking pixel (know when client opens)
- Expiry date untuk quotation (auto-status change)
- Notification ke sales team saat quotation viewed/approved

#### 4.2.3 One-Click Quotation to Project Conversion
**Proses Otomatis:**
1. User klik "Convert to Project" pada approved quotation
2. System creates:
   - **New Project** dengan details dari quotation:
     - Project name, description
     - Client assignment
     - Project timeline/deadline
     - Budget (total quotation value)
   
   - **Task List** based pada quotation line items:
     - Setiap service item → 1 parent task
     - Auto-assign to relevant team based on service category
     - Estimated hours dari quotation → task time budget
     - Task dependencies (optional, based on template)
   
   - **Invoice Schedule**:
     - Sesuai payment terms di quotation
     - Due dates calculated
     - Status: Pending (belum waktunya issue)
   
   - **Client Portal Access**:
     - Auto-generate credentials
     - Send welcome email dengan login link

3. Confirmation dialog menampilkan summary sebelum execute
4. Option untuk customize sebelum finalize

**Benefit:**
- Hemat 2-3 jam setup manual per project
- Zero data entry errors
- Consistency antara quotation dan execution
- Instant project kick-off

**Features:**
- Side-by-side comparison (previous vs current version)

### 4.4 MODUL 4: INVOICING & FINANCE

#### 4.4.1 Invoice Creation

**Automatic Invoice Generation:**
Trigger otomatis saat:
- Milestone project selesai & approved client
- Recurring date untuk retainer project
- Manual trigger oleh admin

**Process:**
1. System ambil data dari:
   - Project details
   - Quotation line items (untuk milestone-based)
   - Time tracking data (untuk hourly-based)
   - Payment schedule yang sudah ditetapkan

2. Generate invoice number (auto-increment dengan format: INV-2024-001)

3. Populate invoice dengan:
   - Client billing information
   - Invoice date & due date
   - Line items dengan descriptions
   - Subtotal, tax, total
   - Payment instructions
   - Terms & conditions

4. Review dialog untuk admin (bisa edit sebelum issue)

5. Issue invoice:
   - Save ke database
   - Generate PDF
   - Send email ke client (dengan PDF attachment)
   - Update client portal

**Manual Invoice Creation:**
Form untuk create invoice dari nol:
- Select client
- Select project (optional, bisa untuk misc. billing)
- Add line items manually
- Set dates & terms
- Preview & send

**Recurring Invoices:**
Untuk retainer clients:
- Set frequency (monthly, quarterly, annually)
- Set start date & end date (atau "until cancelled")
- System auto-generate sesuai schedule
- Preview & approval workflow sebelum send (optional)

#### 4.4.2 Invoice Management Dashboard

**Overview Metrics:**
- **Revenue Cards**:
  - Total revenue (this month, this year)
  - Paid invoices total
  - Outstanding invoices total
  - Overdue amount (dengan alert warna merah)

- **Charts**:
  - Revenue trend (bar chart, monthly)
  - Invoice status distribution (pie chart)
  - Top clients by revenue (horizontal bar)

**Invoice List:**
- Filter by: Status, Client, Date range, Project
- Search by invoice number or client name
- Columns: Number, Client, Date, Due Date, Amount, Status, Actions

- **Draft**: Invoice dibuat tapi belum dikirim

- **Paid**: Pembayaran sudah diterima & verified

#### 4.4.3 Payment Tracking & Reminders

### 5.2 Technology Stack Detail

#### Frontend
**Framework**: React.js (dengan TypeScript)
- **Rationale**: 
  - Large ecosystem & community
  - Rich component libraries (Material-UI, Ant Design, Chakra UI)
  - Strong TypeScript support untuk type safety
  - Excellent developer tooling

**State Management**: Redux Toolkit atau Zustand
- Redux untuk complex state
- Zustand untuk simpler, performant solution

**Styling**: Tailwind CSS + Styled Components
- Tailwind untuk rapid development
- Styled Components untuk dynamic, component-scoped styling

**Form Handling**: React Hook Form + Zod (validation)

**Data Fetching**: React Query (TanStack Query)
- Caching, background refetching
- Optimistic updates
- Infinite scroll

**Real-time**: Socket.io Client
- Real-time notifications
- Live updates pada task boards
- Multi-user collaboration indicators

**Build Tool**: Vite (faster than Create React App)

**Testing**: Jest + React Testing Library + Cypress (E2E)

#### Backend
**Runtime**: Node.js v18+ LTS

**Framework**: Express.js atau Fastify
- Express: Mature, extensive middleware ecosystem
- Fastify: Faster performance, built-in schema validation

**Language**: TypeScript
- Type safety mengurangi runtime errors
- Better IDE support & autocomplete
- Easier refactoring

**API Architecture**: RESTful API (primary) + GraphQL (optional, untuk complex queries)

**Authentication**: 
- JSON Web Tokens (JWT) untuk stateless auth
- Refresh token strategy untuk security
- OAuth 2.0 untuk third-party integrations (future: Google, Slack)

**Authorization**: Role-Based Access Control (RBAC)
- Roles: Owner, Admin, Project Manager, Team Member, Client
- Permissions matrix per resource

**Validation**: Joi atau Zod
- Schema validation untuk semua input
- Type-safe dengan TypeScript

**File Upload**: Multer + Sharp (image processing)

**PDF Generation**: Puppeteer (HTML to PDF)
- Customizable templates
- Support untuk complex layouts

**Email Service**: 
- Nodemailer + SMTP (untuk start)
- SendGrid / AWS SES (untuk scale, better deliverability)

**Scheduled Jobs**: node-cron atau Bull (dengan Redis)
- Recurring invoices
- Payment reminders
- Report generation

**Logging**: Winston atau Pino
- Structured logging
- Multiple transports (console, file, external service)

**Monitoring**:

#### Database
**Primary Database**: PostgreSQL 14+
- **Rationale**:
  - ACID compliance (critical untuk financial data)
  - Rich data types (JSONB untuk flexible schemas)
  - Full-text search capabilities
  - Excellent performance untuk complex queries
  - Strong community & tooling

**Schema Design Highlights**:
- Normalized design (3NF) untuk data integrity
- Indexes pada foreign keys & frequently queried columns
- Partitioning untuk large tables (logs, time tracking)
- Database migrations: Knex.js atau Prisma Migrate

**ORM**: Prisma atau TypeORM
- Prisma: Type-safe, auto-generated client, excellent DX
- TypeORM: More flexible, Active Record pattern

**Caching Layer**: Redis 7+
- Session storage
- Cache frequently accessed data (user permissions, settings)
- Real-time features (pub/sub)
- Rate limiting

**Full-Text Search** (Optional, future): Elasticsearch
- Untuk advanced search features
- Log analytics
